### База данных книжного магазина

Учебный проект по SQL, представляющий собой реляционную базу данных книжного магазина.  
Проект выполнен в рамках практических работ по дисциплине «Управление данными» и охватывает проектирование схемы БД, обеспечение целостности данных и написание SQL-запросов различной сложности.

---

## Цели проекта

- Спроектировать логическую структуру базы данных по заданному предметному описанию
- Реализовать таблицы с ограничениями целостности (PRIMARY KEY, FOREIGN KEY, NOT NULL, UNIQUE, CHECK)
- Заполнить БД тестовыми данными
- Реализовать SQL-запросы различной сложности:
  - `SELECT`, `JOIN`
  - подзапросы
  - `IN / NOT IN`
  - `EXISTS / NOT EXISTS`
  - агрегатные функции
  - группировки и сортировки
  - модифицирующие запросы (`INSERT`, `UPDATE`, `DELETE`)

---

## Структура базы данных

База данных моделирует работу книжного магазина и включает следующие таблицы:

### Покупатель 
- идентификатор
- фамилия
- район проживания
- скидка (%)

### Магазин 
- идентификатор
- название
- район размещения
- комиссионные (%)

### Книги 
- идентификатор
- название
- стоимость
- район складирования
- количество на складе

### Покупка 
- номер заказа
- дата покупки
- продавец
- покупатель
- книга
- количество
- сумма покупки
- комиссионные магазина

Все связи между таблицами реализованы через внешние ключи.

---

## Реализованные задания

В проекте реализованы задания **1–15** из методических указаний, включая:

### Проектирование
- создание таблиц на основе логической схемы
- выбор типов данных
- задание ограничений целостности

### Запросы выборки
- выборка всех записей
- фильтрация по условиям
- сортировка
- поиск по строковым значениям
- работа с датами

### Сложные SQL-конструкции
- подзапросы
- `IN`, `NOT IN`
- `ALL`, `ANY`
- `EXISTS`, `NOT EXISTS`
- `UNION`

### Агрегатные функции
- `COUNT`, `SUM`, `AVG`
- группировка (`GROUP BY`)
- фильтрация агрегатов (`HAVING`)

### Модификация данных
- обновление сумм покупок с учётом скидки
- добавление вычисляемого столбца (комиссионные магазина)
